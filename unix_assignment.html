Unix Basics and Git Practice

Please follow these instructions referencing this guide to get a better understanding of the basics of working with the command line. This will also include some git best practices.

Two of the most basic and practical unix commands are ls and cd. ls will list the contents of your current directory while cd allows you to change your current directory.

  1. In terminal (or other unix command line) enter the command cd ~ to ensure you are at your home directory.
  2. From here, enter the command ls. This will show you all files and directories that exist in your home directory.
  3. Enter the command mkdir unix-test. This will create a new folder in your home directory with the name "unix-test".
  4. Navigate into this new folder with the command cd unix-test.

Navigating the file system this way is very useful when working in a git repository, as you may want to commit only the file changes in an existing directory.

  1. Navigate back to your home directory. Using the above commands, navigate from the home directory to the directory you initialized your git repo in.
  2. Once in your git repo root directory (the directory git init was run in), enter the command git status.
  3. Git status shows you where your local repo is in reference to what was last pushed to your repo. It will list all unpushed commits, all files staged for committing, and all untracked files.
  4. Before working on your repo, it is always best to run git pull or get fetch to ensure you are working with the most recent changes. Run the command git pull.

Now that you know the status of your working directory. Lets make and push a basic change.

  1. In your text editor of choice, create a new .txt file in your repo named "unix-basics.txt".
  2. Referencing the unix cheat sheet, see if you can figure out how to add text to the file from the command line. If you are stuck, discuss with your fellow classmates before asking me for help.
  3. Once you have added text to the file, from the command line run git status. You should now see yout .txt file as an untracked change.
  4. Run the command git add . followed by the command git status. You should now see that the new file has transitioned from untracked to staged.

Now that you have made changes, we can commit and push these changes to your remote repo.

  1. From the command line, you will now run git commit -m "message here". Instead of "message here", please enter a messsage you think briefly describes the change you are making.
  2. Run the command git status once again to ensure that you have a new commit to be pushed to your remote repository.
  3. Run the command git push to push your code to the remote. If you get any kind of feedback at this stage that the code was not pushed successfully, please let me know and I will assist you.
